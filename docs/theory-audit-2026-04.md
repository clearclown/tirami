# Tirami 理論監査メモ 2026-04-27

このメモは、公開前の方向性を冷静に判断するための作業仮説である。参照した外部リポジトリは `tirami-economics`、`tirami-bank`、`tirami-agora`、`tirami-mind`。`bank`、`agora`、`mind` は Python scaffold がアーカイブされ、Rust ワークスペース側に統合済みという位置づけだった。

## 結論

Tirami の方向性は強い。ただし、最初から「世界通貨」や「完全な自律経済」として出すべきではない。公開時の主張は「暗号化 P2P 上で、余っている計算資源を AI 推論に使い、内部会計単位 TRM で精算するプロトコル」に絞るべきである。

経済学的には、TRM は最初から貨幣ではなく、まずは計算の会計単位である。貨幣の三機能、すなわち交換手段、価値尺度、価値保存をすべて満たすには、検証可能性、品質評価、流動性、法的整理、継続的な需要が必要になる。初期段階では「価値尺度」と「内部精算」に集中する方が堅い。

2026-04-26 の Tailscale 2台実験により、最低限の
private-alpha 条件は一段進んだ。Mac Studio と ASUS ROG X13
の間で、agent が peer hint なしに provider を自動選択し、
推論結果、TRM 支出、TRM 収益、両側台帳、再起動後復元まで
確認できた。ただし、これは「需要がある」ことの証明ではなく、
「需要を測れる最小ネットワークが動いた」という証明である。

## 歴史的位置づけ

エネルギーや計算を価値の基礎に置く発想は新しくない。Soddy のエネルギー経済、Technocracy movement のエネルギー証書、Buckminster Fuller の kWh 通貨、Bitcoin の proof-of-work は同じ系譜にある。

Tirami の差分は、計算を無目的に燃やすのではなく、AI 推論という有用な仕事に向ける点にある。この差分は大きいが、同時に難所も増える。Bitcoin は SHA-256 の検証が単純だったから成立した。Tirami は「その推論が本当に行われたか」「品質が十分か」「需要者が満足したか」を扱う必要がある。

したがって、Tirami の歴史的な比較対象は Bitcoin だけではない。SETI@home や Folding@home のボランタリ計算、クラウドの spot instance、市場型クラウド、Bittensor、Akash、Nostr/NIP-90 系のジョブ市場とも比較すべきである。

## 経済学的評価

需要は十分あり得る。ローカル推論、プライバシー、遊休 PC、企業内の未使用計算資源、学習・研究用途、低単価バッチ推論には現実的な需要がある。

ただし、需要が「絶対に世界中で大きい」と言い切れるのは、実測後である。市場は思想ではなく、価格、速度、信頼性、導入摩擦、セキュリティで決まる。公開前に必要なのは主張の強化ではなく、測定可能な比較である。

主な経済リスクは次の通り。

- 逆選択: 低品質・低信頼ノードほど参加しやすい。
- モラルハザード: 計算をしたふり、低品質な出力、途中切断が起きる。
- Sybil/談合: ノード数や取引量を偽装できる。
- 価格形成: FLOP 単価だけでは、メモリ帯域、モデル品質、遅延、電力、失敗率を反映しきれない。
- 外部性: ノード所有者の電力代、熱、バッテリー劣化、ネットワーク負荷を誰が負担するか。
- 法務・規制: TRM を販売・投機対象に見せると、プロトコルより金融商品として扱われるリスクが増える。

## 実装の優先順位

公開準備では Layer 2 以降を前面に出しすぎない方がよい。Bank、Agora、Mind は魅力的だが、土台になる推論・精算・評判が安定していなければ、金融機能だけが先に膨らむ。

優先順位は次の順が妥当である。

1. L0/L1: 暗号化 P2P、推論、署名付き取引、永続台帳、復旧。
2. Private testnet: 2-4台で、実ジョブが発生し、TRM の earned/spent が増えること。2026-04-26 に2台で達成済み。
3. Stress test: 10ノード以上、7日以上、切断・再接続・失敗時精算を検証。
4. Agora: provider discovery、品質評価、評判、ジョブ市場。
5. Mind: agent が予算内で local/remote を選び、支出と収益を説明可能にする。
6. Bank: レンディング、保険、先物は最後。実需と失敗率が測定されてからでよい。

## 公開時の言い方

避けるべき表現:

- 「TRM は投資対象」
- 「世界通貨」
- 「絶対に儲かる」
- 「完全自律経済が完成」

使うべき表現:

- 「TRM は計算の内部会計単位」
- 「1 TRM = 検証・計量された計算量を表すプロトコル上の単位」
- 「公開初期は実験的 testnet」
- 「投機ではなく、推論ジョブの精算に使う」
- 「未監査部分、未実装部分、既知の制約を明示する」

## 直近の判定基準

実用可能と呼ぶ最低ラインは、以下を満たすことである。

- 2台以上の実機で P2P peer として相互認識する。
- 片方の agent がもう片方へ推論タスクを投げられる。
- provider 側の台帳に trade が記録される。
- consumer 側の agent に spent、provider 側の agent に earned が記録される。
- ノード再起動後も identity、ledger、agent state が維持される。
- HTTP API を公開する場合は bearer token が必須。
- README と SECURITY に「できること」「まだできないこと」が明記される。

2026-04-26 の実験でこの最低ラインの多くは満たした。したがって
「private alpha testnet として公開準備に入れる」は妥当である。
一方、「実用可能で完成」や「open public testnet 健全」と呼ぶには
まだ早い。次の客観基準は、10ノード以上、7日以上、毎日1件以上の
実ジョブ、再起動復元、失敗時精算、ログ/メトリクス監視を満たすこと。
