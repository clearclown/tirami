# Forge — ארכיטקטורה (Architecture)

## סקירה כללית (Overview)

מערכת Forge היא מערכת דו-שכבתית: **הסקה (inference)** ו-**כלכלה (economy)**.

שכבת ההסקה מטפלת בהפצת מודלים, רשת מש (mesh networking), והגשת API. היא בנויה על בסיס [mesh-llm](https://github.com/michaelneale/mesh-llm).

שכבת הכלכלה מטפלת בחשבונאות CU, רישום עסקאות, תמחור ותקציבי סוכנים (agent budgets). זוהי התרומה המקורית של Forge.

```
┌─────────────────────────────────────────────────┐
```

## שכבת ההסקה (mesh-llm)

שכבת ההסקה אחראית על:

- **רשת מש (Mesh networking)**: חיבורי QUIC מבוססי iroh עם הצפנת Noise.
- **גילוי עמיתים (Peer discovery)**: ממסרי Nostr עבור רשתות מש ציבוריות, mDNS עבור רשת מקומית (LAN).
- **הפצת מודלים (Model distribution)**: מקביליות צינור (Pipeline parallelism) עבור מודלים דחוסים, שיתוף מומחים (expert sharding) עבור MoE.
- **ביצוע הסקה (Inference execution)**: llama.cpp באמצעות תהליכי משנה של llama-server ו-rpc-server.
- **הגשת API**: תואם OpenAI בנתיבים `/v1/chat/completions` ו-`/v1/models`.

Forge יורשת את כל זה מ-mesh-llm. שכבת ההסקה אינה מודעת ל-CU, עסקאות או תמחור.

## השכבה הכלכלית (Forge)

השכבה הכלכלית יושבת מעל ההסקה ואחראית על:

### forge-ledger — המנוע הכלכלי

```rust
pub struct ComputeLedger {
    balances: HashMap<NodeId, NodeBalance>,
    work_log: Vec<WorkUnit>,
    trade_log: Vec<TradeRecord>,
    price: MarketPrice,
}
```

אחריות ליבה:
- מעקב אחר יתרת CU לכל צומת (תרומה, צריכה, הזמנה).
- רישום כל עסקת הסקה (ספק, צרכן, כמות CU, טוקנים).
- חישוב מחירי שוק דינמיים לפי היצע וביקוש.
- החלת תשואה (Yield) על צמתים תורמים.
- ייצוא דוחות סליקה עבור גשרים מחוץ לפרוטוקול.
- שמירת תצלומי מצב (snapshots) לדיסק עם הגנת שלמות HMAC-SHA256.

### forge-verify — הוכחת עבודה מועילה (יעד)

מבטיחה שתביעות CU הן לגיטימיות:
- פרוטוקול חתימה כפולה (Dual-sign): גם הספק וגם הצרכן חותמים על כל TradeRecord.
- סנכרון Gossip: עסקאות חתומות מופצות ברחבי הרשת.
- אימות: כל צומת יכול לאמת את שתי החתימות.
- זיהוי הונאה: עסקאות לא תואמות או לא חתומות נדחות.

### forge-bridge — סליקה חיצונית (אופציונלי)

ממיר יחידות CU לערך חיצוני עבור מפעילים שזקוקים לכך:
- Bitcoin Lightning: המרת CU ← msats באמצעות שער חליפין שניתן להגדרה.
- מטבע יציב (Stablecoin): המרת CU ← USDC/USDT באמצעות מתאם (adapter).
- פיאט (Fiat): המרת CU ← העברה בנקאית באמצעות דשבורד המפעיל.

שכבת הגשר נמצאת מחוץ לפרוטוקול הליבה. מפעילים שונים יכולים להשתמש בגשרים שונים.

### ממשק API

| נתיב | שכבה | תיאור |
|-------|-------|-------------|
| `POST /v1/chat/completions` | הסקה + כלכלה | ביצוע הסקה, רישום עסקת CU |
| `GET /v1/models` | הסקה | רשימת מודלים טעונים |
| `GET /v1/forge/balance` | כלכלה | יתרת CU, מוניטין |
| `GET /v1/forge/pricing` | כלכלה | מחיר שוק, הערכות עלות |

## זרימת נתונים (Data Flow)

### הסקה עם חשבונאות CU

```
הצרכן שולח בקשה
    ↓
ה-API מקבל POST /v1/chat/completions
    ↓
הספר (Ledger) בודק: האם הצרכן יכול לשלם (can_afford)?
    ↓ כן
שכבת ההסקה מתבצעת (llama-server / rpc-server)
    ↓
טוקנים מוזרמים בחזרה לצרכן
    ↓
הספר רושם את העסקה:
  - provider.contributed += cu_cost
  - consumer.consumed += cu_cost
  - trade_log.push(TradeRecord)
    ↓
התגובה כוללת x_forge: { cu_cost, effective_balance }
```

### ייצוא סליקה

```
המפעיל מריץ: forge settle --hours 24
    ↓
ה-API קורא את trade_log עבור חלון הזמן
    ↓
סיכום לכל צומת: סך שהורווח, סך שהוצא, נטו CU
    ↓
ייצוא דוח JSON עם מחיר ייחוס אופציונלי
    ↓
המפעיל משתמש במתאם גשר להמרת נטו CU ל-BTC/פיאט
```

## מודל אבטחה

```
שכבה 0: ביטקוין mainchain ← עיגון אופציונלי (בעתיד)
שכבה 1: חתימות כפולות   ← ספק + צרכן חותמים על כל עסקה
שכבה 2: ספר HMAC-SHA256 ← הגנת שלמות מקומית
שכבה 3: iroh (QUIC+Noise)← הצפנת תעבורה
שכבה 4: ביצוע הסקה       ← המודל רץ מקומית אצל הספק
```

כל שכבה מגנה מפני איומים שונים:
- שכבה 4: שלמות המודל (אימות האש GGUF).
- שכבה 3: סודיות תעבורה (האזנה).
- שכבה 2: שיבוש מקומי (שינוי קבצים).
- שכבה 1: הונאת רשת (תביעות CU מזויפות).
- שכבה 0: חוסר משתנות היסטורי (עיגון אופציונלי לביטקוין).

## תלויות Crates

```
forge-core ← סוגים משותפים (NodeId, CU, Config)
    ↑
forge-ledger ← מנוע כלכלי (עסקאות, תמחור, תשואה)
    ↑
forge-lightning ← גשר חיצוני (ארנק LDK, המרת CU↔sats)
    ↑
forge-node ← תזמור (HTTP API, צינור, אינטגרציית ספר)
    ↑
forge-cli ← ממשק שורת פקודה (chat, seed, worker, settle)

forge-net ← תעבורת P2P (iroh, QUIC, Noise, mDNS)
forge-proto ← הודעות רשת (bincode, 14 payload types)
forge-infer ← מנוע הסקה (llama.cpp, GGUF loader)
forge-shard ← מתכנן טופולוגיה (הקצאת שכבות, איזון מחדש)
```
