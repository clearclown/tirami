# Forge — רצף אתחול (Bootstrap Sequence)

## סקירה כללית (Overview)

ל-Forge יש שני נתיבי אתחול:

- **זרימת הייחוס הנוכחית (current reference flow)**, שהיא מפורשת ומונעת על ידי המפעיל.
- **הזרימה המיועדת (target flow)**, שבה צומת מבוסס mesh-llm מצטרף לרשת ומתחיל להרוויח CU באופן אוטומטי.

## זרימת הייחוס הנוכחית (Current Reference Flow)

```text
1. הפעלת מארח מודל: forge seed -m "qwen2.5:0.5b"
2. העתקת המפתח הציבורי המודפס
3. חיבור מבקש (Requester): forge worker --seed <seed_public_key>
4. בדיקת סטטוס: forge status --url http://127.0.0.1:3000
5. בדיקת יתרת CU: curl http://127.0.0.1:3000/v1/forge/balance
```

ממשק ה-HTTP API נקשר ל-`127.0.0.1` כברירת מחדל. אם הוא חשוף לרשת, הגדר `--api-token`.

## אתחול מיועד (Target Bootstrap - פיצול mesh-llm)

ברגע ש-Forge ישתלב עם mesh-llm:

```text
1. forge --auto                          # הצטרפות לרשת הציבורית הטובה ביותר
2. forge --model Qwen2.5-32B --publish   # יצירת רשת ציבורית, הרווחת CU
3. forge --join <token>                  # הצטרפות עם GPU, הרווחת CU
4. forge --client --join <token>         # הצטרפות כצרכן, הוצאת CU
```

כל הסקה המוגשת מרוויחה CU. כל הסקה שנצרכת עולה CU. השכבה הכלכלית היא אוטומטית — אין צורך בהגדרה נפרדת.

## אתחול כלכלי (Economic Bootstrap)

### צומת חדש (יתרה אפס)

```text
1. צומת מצטרף לרשת
2. שכבה חינמית: 1,000 CU זמינים מיידית
3. צומת מגיש בקשת הסקה ראשונה ← מרוויח CU
4. יתרת ה-CU גדלה עם כל בקשה המוגשת
5. הצומת יכול כעת להוציא CU על הסקה של צמתים אחרים
```

### צומת קיים (בעל יתרה)

```text
1. צומת מאתחל מחדש, טוען ספר חשבונות שמור (forge-ledger.json)
2. שלמות HMAC-SHA256 מאומתת
3. יתרה קודמת, עסקאות ומוניטין משוחזרים
4. הצומת ממשיך להרוויח ולהוציא CU
```

## הידרדרות והתאוששות (Degradation & Recovery)

| אירוע | השפעה כלכלית | השפעת הסקה |
|---|---|---|
| צומת מרוחק אחד מתנתק | הצמתים הנותרים סופגים את העבודה, זרימת CU נמשכת | הפסקה קצרה, המודל מאוזן מחדש |
| כל הצמתים המרוחקים מתנתקים | כלכלת CU עוצרת, מצב מקומי בלבד | חזרה למודל מקומי קטן |
| סוללת צומת נמוכה (<20%) | הפסקת הגשה (הרווחה עוצרת), עדיין ניתן לצרוך | העברת שכבות לצמתים מרוחקים |
| צומת חוזר לרשת | המשך הרווחת CU, המוניטין מתאושש | גילוי מחדש של עמיתים, הרחבה מחדש |

**קבוע ליבה**: יתרת ה-CU של צומת נשמרת לאורך אתחולים וניתוקים. CU שהורווח לעולם אינו אובד.

## מודל תרומת צומת (Node Contribution Model)

- **תורמים (Contributors)**: מכשירים המגישים הסקה מרוויחים CU.
- **צרכנים (Consumers)**: מכשירים המבקשים הסקה מוציאים CU.
- **יתרה**: יותר תרומה ← יותר CU ← יותר גישה למחשוב של אחרים.
- **שכבה חינמית**: 1,000 CU לצמתים חדשים, נצרכים מהבקשה הראשונה.
- **תשואה (Yield)**: צמתים מחוברים מרוויחים 0.1% תשואה לשעה (משוקלל מוניטין).
- **ללא תשלום חובה**: הפרוטוקול רץ על CU. גשרים חיצוניים (Lightning, פיאט) הם אופציונליים.

## אבטחה במהלך האתחול

- זהות Ed25519 נוצרת לפני כל פעילות רשת.
- כל החיבורים מוצפנים באמצעות QUIC + Noise.
- בזרימת seed/worker הנוכחית, ה-seed רואה את טקסט הבקשה (גבול אמון מפורש).
- עסקאות CU נרשמות מקומית עם הגנת שלמות HMAC-SHA256.
- יעד: עסקאות בחתימה כפולה המסונכרנות באמצעות Gossip ברחבי הרשת.
