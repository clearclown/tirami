# Forge — מפת דרכים (Roadmap)

## שלב 1: הסקה מקומית ✅

- `forge-core`: מערכת סוגים (NodeId, LayerRange, ModelManifest, PeerCapability)
- `forge-infer`: מנוע llama.cpp, טוען GGUF, ייצור טוקנים בהזרמה
- `forge-node`: ממשק HTTP API (/chat, /chat/stream, /health)
- `forge-cli`: פקודת `forge chat` עם הורדת מודל אוטומטית

## שלב 2: פרוטוקול P2P ✅

- `forge-net`: תעבורת Iroh, הצפנת Noise, חיבורי עמיתים
- `forge-proto`: 14 סוגי הודעות בפרוטוקול (bincode + קידומת אורך)
- `forge-node`: צינור Seed/Worker, בקשת/תגובת הסקה
- בדיקות אינטגרציה: שני צמתים מחליפים הודעת Hello + הודעות מרובות

## שלב 3: הסקה מרוחקת + ספר חשבונות מפעיל ✅

- `forge-ledger`: חשבונאות CU, ביצוע עסקאות, מוניטין, תשואה, תמחור שוק
- `forge-node`: ספר חשבונות משולב בצינור ההסקה
- בדיקות יתרת CU לפני הסקה
- רישומי עסקאות לאחר סיום
- שלמות ספר חשבונות HMAC-SHA256

## שלב 4: ממשק API כלכלי ✅

- ממשק API תואם OpenAI: `POST /v1/chat/completions`, `GET /v1/models`
- מדידת CU: כל הסקה רושמת עסקה עם סיומת `x_forge`
- נקודות קצה לתקציב סוכנים: `GET /v1/forge/balance`, `GET /v1/forge/pricing`
- גשר סליקה CU→Lightning: הפקודה `forge settle --pay`
- זיהוי אוטומטי של מודל Seed מ-HF Hub
- סגירה מסודרת (Ctrl-C) עם שמירת ספר החשבונות

## שלב 5: שילוב פיצול mesh-llm (הבא)

**יעד:** החלפת שכבת ההסקה של Forge במנוע המבוזר המוכח של mesh-llm.

| תוצר | תיאור |
|---|---|
| פיצול mesh-llm | יצירת forge כפיצול של mesh-llm עם שכבה כלכלית |
| שילוב forge-ledger | חיבור רישום CU לצינור ההסקה של mesh-llm |
| שימור API כלכלי | שמירה על נקודות קצה של /v1/forge/* בקוד החדש |
| הרחבת קונסולת אינטרנט | הוספת יתרת CU ותצוגת עסקאות לקונסולת mesh-llm |
| Pipeline + MoE | ירושת מקביליות צינור ושיתוף מומחים מ-mesh-llm |
| גילוי Nostr | ירושת גילוי רשת מש ציבורית מ-mesh-llm |
| CREDITS.md | תיעוד ייחוס ל-mesh-llm |

## שלב 6: הוכחת עבודה מועילה (Proof of Useful Work)

**יעד:** הפיכת תביעות CU למאומתות ברחבי הרשת.

| תוצר | תיאור |
|---|---|
| פרוטוקול חתימה כפולה | גם הספק וגם הצרכן חותמים על כל TradeRecord |
| סנכרון Gossip | עסקאות חתומות מופצות ברחבי הרשת |
| זיהוי הונאה | דחיית עסקאות לא חתומות או לא תואמות |
| הפצת מוניטין | שיתוף ציוני מוניטין בין עמיתים |
| עמידות לקנוניה | זיהוי אנומליות סטטיסטיות בדפוסי עסקאות |

## שלב 7: גשרים חיצוניים

**יעד:** מתן אפשרות למפעילים להמיר CU לערך חיצוני.

| תוצר | תיאור |
|---|---|
| גשר Lightning | סליקה אוטומטית CU→sats דרך LDK |
| מתאם מטבע יציב | המרת CU→USDC/USDT |
| ממשק מתאם פיאט | מפרט לסליקה בהעברה בנקאית |
| שירות שער חליפין | עדכוני שער ציבוריים עבור CU/BTC ו-CU/USD |
| עיגון לביטקוין | אופציונלי: שורש Merkle תקופתי ← OP_RETURN לתיעוד ביקורת בלתי משתנה |

## שלב 8: כלכלה אוטונומית של סוכנים

**יעד:** מתן אפשרות לסוכני AI לנהל את מחזור חיי המחשוב שלהם בעצמם.

| תוצר | תיאור |
|---|---|
| מדיניות תקציב | מגבלות הוצאה שנקבעות על ידי אדם לכל סוכן |
| מסחר אוטונומי | הסוכן מחליט מתי לקנות/למכור מחשוב |
| ניתוב רב-מודלים | הסוכן בוחר מודל על בסיס פשרה בין עלות לאיכות |
| חיזוק עצמי | הסוכן מרוויח CU ← קונה גישה למודל גדול יותר ← מרוויח עוד CU |
| כלכלה בין-סוכנית | סוכנים סוחרים במחשוב מתמחה (מודל קוד לעומת מודל צ'אט) |

## טווח ארוך

| אבן דרך | תיאור |
|---|---|
| שחרור SDK | forge-node כספריית Rust להטמעה עם API יציב |
| פרוטוקול גרסה 2 | לקחים מגרסה 1, אבולוציה תואמת לאחור |
| ריבוי ארכיטקטורות | תמיכה ב-NVIDIA GPU, AMD ROCm, RISC-V (דרך mesh-llm) |
| אימון פדרטיבי | כוונון עדין (fine-tuning) מבוזר, לא רק הסקה |
| נגזרי מחשוב | חוזים עתידיים על קיבולת מחשוב עתידית |

> הפרוטוקול הוא הפלטפורמה. המחשוב הוא המטבע.
